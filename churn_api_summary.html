What it does

The problem
Companies lose 5–25% revenue annually to churn. Teams have no way to know which customers are at risk until they have already left. Models also silently degrade in production without anyone noticing.
The solution
A REST API any app can call to score a customer in real time. Returns a churn probability, risk tier (low/medium/high), and the top 3 reasons why — derived from model coefficients, so no extra library needed.

Core features

01
Real-time churn scoring
POST /predict — send customer data, get back a probability (0–100%), a risk tier, and the model version that scored it.
02
Coefficient-based explainability updated
Every prediction includes the top 3 features driving the score — e.g. "month-to-month contract (+0.31)" — computed directly from LR coefficients × feature values. No SHAP library needed.
03
MLflow experiment tracking
Every training run logs hyperparameters, AUC, F1 to MLflow. Compare any two runs side by side in the MLflow UI.
04
MLflow Model Registry
Models are versioned through Staging → Production. The API always loads from Production — no manual file copying needed.
05
On-demand retraining
POST /retrain triggers a full retrain, compares new AUC vs the current champion in MLflow, and promotes only if it genuinely improves by > 0.005.
06
Swagger UI + health check
Auto-generated interactive docs at /docs. GET /health confirms model is loaded and ready. GET /model/info shows active version and metrics.
